# 🚀 Применение улучшений поиска по номенклатуре

## Быстрый старт

### 1. Переиндексация с улучшенными настройками

```bash
cd server
npm run reindex:improved
```

### 2. Тестирование улучшений

```bash
npm run test:search
```

## Что изменилось

✅ **Увеличен boost для точных совпадений** - марки бетона (В25, М300) получают максимальный приоритет

✅ **Исключены растворы из синонимов бетона** - поиск "В25" не покажет растворы

✅ **Добавлены специальные синонимы** - поддержка различных форматов записи марок

✅ **Улучшен function_score** - дополнительный бустинг для документов с марками бетона

✅ **Специальная логика для марок** - отдельная обработка запросов типа "В25", "М300"

## Ожидаемые результаты

- **Поиск "В25"** → показывает только бетон В25 на первых позициях
- **Поиск "раствор"** → показывает только растворы, не бетон
- **Высокие scores** для релевантных результатов
- **Минимум нерелевантных** позиций в выдаче

## Мониторинг

После применения проверьте:

1. Результаты поиска по маркам бетона
2. Отсутствие растворов в результатах поиска бетона
3. Высокие scores для точных совпадений

## Откат изменений

Если нужно откатить изменения:

1. Восстановите старые настройки в `elasticsearchService.ts`
2. Переиндексируйте данные: `npm run reindex:bitrix`
