# Database Manager - Управление базой данных

Компонент для управления базой данных с возможностью редактирования в реальном времени.

## Основные возможности

### 🔧 Система нумерации полей

Введена четкая система нумерации с автоматической валидацией:

#### Правила нумерации:

- **Разделы (headers)**: 100, 200, 300, 400...
- **Поля в разделах**: 101, 102, 103... (для раздела 100), 201, 202, 203... (для раздела 200)
- **Поля без раздела**: 1, 2, 3, 4...

#### Автоматическая нормализация:

- Кнопка "Нормализовать порядок" автоматически исправляет нумерацию
- Предварительный просмотр изменений перед применением
- Валидация при ручном редактировании

### 📊 Колонки таблицы "Поля форм"

1. **Название поля** - техническое имя поля
2. **Отображаемое название** - видимый пользователю текст
3. **Тип** - тип поля (text, select, header, etc.)
4. **Порядок** - номер для сортировки (с валидацией)
5. **Раздел** - к какому разделу принадлежит поле
6. **Обязательное** - требуется ли заполнение
7. **Битрикс ID** - идентификатор в Битрикс24

### 🎯 Inline редактирование

- **Double Click** - начать редактирование ячейки
- **Enter** - сохранить изменения
- **Escape** - отменить редактирование
- **Автосохранение** при потере фокуса

### 🔄 Обновление данных

После каждого изменения:

1. Валидация значения (для поля "Порядок")
2. Отправка на сервер
3. Автоматическое обновление таблицы
4. Уведомление о результате

## Архитектура

### Компоненты

```
DatabaseManager/
├── index.tsx                     # Главный компонент с вкладками
├── components/
│   └── DatabaseTable.tsx        # Универсальная таблица
├── hooks/
│   ├── useFormFields.ts         # Логика работы с полями
│   └── useUsers.ts              # Логика работы с пользователями
└── utils/
    ├── sectionUtils.ts          # Утилиты для работы с разделами
    └── orderNormalizer.ts       # Нормализация порядка полей
```

### Утилиты разделов

#### `sectionUtils.ts`

- `isSection()` - проверка типа "header"
- `getSectionNumber()` - определение номера раздела
- `getFieldSectionName()` - получение названия раздела для поля
- `validateFieldOrder()` - валидация порядка согласно правилам

#### `orderNormalizer.ts`

- `calculateNormalizedOrder()` - расчет правильной нумерации
- `generateNormalizationReport()` - отчет о планируемых изменениях

### Хуки

#### `useFormFields`

```typescript
const { data, isLoading, updateField, error } = useFormFields()
```

- **data** - массив полей формы
- **updateField** - функция обновления с валидацией
- **isLoading** - состояние загрузки
- **error** - ошибки

## Использование

### Редактирование порядка

1. Дважды кликните на ячейку "Порядок"
2. Введите новое значение
3. Нажмите Enter или уберите фокус
4. Система автоматически валидирует значение

### Нормализация порядка

1. Нажмите кнопку "Нормализовать порядок"
2. Просмотрите предлагаемые изменения
3. Нажмите "Применить изменения"
4. Дождитесь завершения процесса

### Понимание разделов

- Колонка "Раздел" показывает принадлежность поля
- Для headers: "РАЗДЕЛ: Название"
- Для обычных полей: название родительского раздела
- Для полей без раздела: "Без раздела"

## Валидация

### Правила для разделов (headers)

- Порядок должен быть кратен 100
- Автоматическое предложение ближайшего корректного значения

### Правила для обычных полей

- Порядок должен быть в пределах раздела
- Для раздела 100: поля могут иметь порядок 101-199
- Автоматическое предложение следующего доступного номера

## Обработка ошибок

- Валидация на клиенте перед отправкой
- Откат изменений при ошибке сервера
- Детальные сообщения об ошибках
- Автоматическое обновление после успешного сохранения

## Производительность

- Мемоизация конфигурации колонок
- Оптимистичные обновления
- Debounced поиск
- Ленивая загрузка больших таблиц

## Безопасность

- Валидация всех входящих данных
- Проверка прав доступа на сервере
- Санитизация пользовательского ввода
- Защита от XSS атак
