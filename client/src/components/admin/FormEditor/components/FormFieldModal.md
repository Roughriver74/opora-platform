# FormFieldModal

Модальное окно для редактирования полей формы, заменяющее старый аккордеон-интерфейс.

## Особенности

### ✅ Модальный интерфейс

- Полноэкранное модальное окно для удобного редактирования
- Правильная фокусировка и навигация
- Закрытие по Escape или клику вне окна

### ✅ Улучшенный UX

- Четкая структура с разделами настроек
- Валидация полей с понятными ошибками
- Интуитивный интерфейс с группировкой типов полей

### ✅ Полная функциональность

- Все типы полей поддерживаются
- Интеграция с Битрикс24
- Настройки динамических источников
- Работа с вариантами ответов
- Связанные поля

## Использование

```tsx
import FormFieldModal from './FormFieldModal'

function MyComponent() {
	const [editingField, setEditingField] = useState<FormField | null>(null)

	return (
		<FormFieldModal
			open={editingField !== null}
			field={editingField || {}}
			onSave={field => {
				// Сохранить поле
				setEditingField(null)
			}}
			onClose={() => setEditingField(null)}
			onDelete={() => {
				// Удалить поле
				setEditingField(null)
			}}
			availableBitrixFields={bitrixFields}
			allFields={formFields}
		/>
	)
}
```

## Пропсы

### Обязательные

- `open: boolean` - Открыто ли модальное окно
- `field: Partial<FormField>` - Данные редактируемого поля
- `onSave: (field: Partial<FormField>) => void` - Callback сохранения
- `onClose: () => void` - Callback закрытия
- `availableBitrixFields: Record<string, any>` - Доступные поля Битрикс

### Опциональные

- `onDelete?: () => void` - Callback удаления поля
- `allFields?: FormField[]` - Все поля формы (для выбора разделов)

## Поддерживаемые типы полей

### Структурные элементы

- `header` - Заголовок раздела
- `divider` - Разделитель

### Поля ввода

- `text` - Текстовое поле
- `textarea` - Многострочный текст
- `number` - Числовое поле
- `date` - Поле даты

### Поля выбора

- `select` - Выпадающий список
- `autocomplete` - Автозаполнение
- `radio` - Переключатели
- `checkbox` - Флажки

## Валидация

### Обязательные поля

- Идентификатор поля (name)
- Название поля (label)

### Автоматические проверки

- Уникальность идентификатора
- Корректность типа поля
- Валидность опций для полей выбора

## Интеграция с Битрикс24

### Автоматическая загрузка

- Подключение к полям сделок Битрикс
- Загрузка вариантов для enum-полей
- Синхронизация типов данных

### Настройка связи

1. Выберите поле Битрикс из списка
2. Система автоматически загрузит варианты (если есть)
3. Настройте дополнительные параметры

## Динамические источники

### Настройка

- Включите опцию "Использовать динамический источник"
- Укажите источник данных (API endpoint или поле)
- Настройте фильтры при необходимости

### Применение

- Данные загружаются динамически при заполнении формы
- Поддержка кэширования и обновления
- Обработка ошибок загрузки

## Связанные поля

### Автоматическое копирование

- Настройка источника для копирования значений
- Выбор направления копирования
- Трансформация данных при копировании

### Конфигурация

1. Перейдите к разделу "Связанные поля"
2. Выберите поле-источник
3. Настройте правила копирования

## Клавиатурные сокращения

- `Escape` - Закрыть модальное окно
- `Enter` - Сохранить (когда фокус на кнопке)
- `Tab` - Навигация между полями
