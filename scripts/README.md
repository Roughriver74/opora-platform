# Скрипты Beton CRM

## Основные скрипты

### Запуск и остановка

- `./start.sh` - Запуск в production режиме (собранные контейнеры)
- `./start-dev.sh` - Запуск в режиме разработки (с hot reload)
- `./stop.sh` - Остановка всех контейнеров
- `./logs.sh` - Просмотр логов (можно указать сервис: backend, frontend, db, redis)

### Резервное копирование

- `./backup.sh` - Создание резервной копии БД и файлов
- `./restore.sh` - Восстановление из резервной копии

### Миграция данных

- `./migrate.sh` - Миграция данных из MongoDB в PostgreSQL

## Использование

```bash
# Запуск в production
cd scripts
./start.sh

# Запуск в dev режиме
./start-dev.sh

# Просмотр логов backend
./logs.sh backend

# Остановка
./stop.sh
```

## Старые скрипты

Все старые и неиспользуемые скрипты перемещены в `backup-old-scripts/`

## Обслуживание

- `maintenance/cleanup-disk.sh` — локальная/серверная очистка Docker-кеша, логов контейнеров, старых бэкапов и временных файлов. Поддерживает режим `--dry-run` и настраиваемые пороги через переменные окружения.
- `maintenance/setup-cleanup-cron.sh` — копирует скрипт очистки на удалённый сервер и создает cron-задание (по умолчанию раз в час). Использует SSH/ SCP и позволяет задать расписание и пути через переменные.
