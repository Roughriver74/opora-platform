# Бетон CRM - Система оформления заявок на покупку бетона

Веб-приложение для создания и управления формами заявок на покупку бетона с интеграцией Битрикс24.

## Функциональность

- Динамические формы заявок с различными типами полей (текстовые, выпадающие списки, чекбоксы и т.д.)
- Интеграция с Битрикс24 для создания сделок
- Выпадающие списки с автоматическим заполнением из номенклатуры Битрикс24
- Админ-панель для настройки полей формы
- Возможность создания нескольких форм

## Технический стек

- **Frontend**: React.js с TypeScript, Material-UI
- **Backend**: Node.js, Express.js, TypeScript
- **База данных**: MongoDB
- **Интеграция**: Битрикс24 REST API

## Установка и запуск

### Предварительные требования

- Node.js (версия >= 14)
- MongoDB (установленная и запущенная)

### Установка зависимостей

1. Клонировать репозиторий:

```bash
git clone <url-репозитория>
cd beton-crm
```

2. Установить зависимости для серверной части:

```bash
cd server
npm install
```

3. Установить зависимости для клиентской части:

```bash
cd ../client
npm install
```

### Настройка окружения

1. В директории `server` создайте файл `.env` или отредактируйте существующий:

```
PORT=5000
MONGODB_URI=mongodb://localhost:27017/beton-crm
BITRIX24_WEBHOOK_URL=https://crm.betonexpress.pro/rest/3/74sbx907svrq1v10/
```

2. В директории `client` создайте файл `.env` или отредактируйте существующий:

```
REACT_APP_API_URL=http://localhost:5000/api
```

### Запуск приложения

1. Запустите сервер (в директории `server`):

```bash
npm run dev
```

2. В отдельном терминале запустите клиентское приложение (в директории `client`):

```bash
npm start
```

3. Откройте браузер и перейдите по адресу: [http://localhost:3000](http://localhost:3000)

## Использование

### Публичная часть (форма заказа)

- Главная страница содержит форму заказа бетона
- Заполнение формы и нажатие кнопки "Отправить заявку" создает сделку в Битрикс24

### Административная часть

- Перейдите на страницу администрирования: [http://localhost:3000/admin](http://localhost:3000/admin)
- Здесь можно создавать и настраивать формы
- Можно указать поля из Битрикс24, которые будут использоваться при создании сделок
- Для полей типа "выпадающий список" или "автозаполнение" можно выбрать источник данных из номенклатуры Битрикс24

## Деплой приложения

Для развертывания приложения на сервере используется автоматический скрипт:

```bash
# ВАЖНО: скрипт необходимо запускать из корневой директории проекта
cd /Users/evgenijsikunov/projects/beton-crm/beton-crm && ./scripts/manual-deploy.sh
```

Скрипт выполнит следующие действия:

1. Сохранит резервную копию файла `.env`
2. Установит необходимые настройки для продакшн-сборки
3. Соберёт клиентскую и серверную части приложения
4. Восстановит исходный `.env` файл
5. Создаст архив с собранными файлами
6. Загрузит архив на сервер
7. Распакует файлы и перезапустит приложение

Более подробные инструкции по развертыванию можно найти в файле [DEPLOYMENT.md](DEPLOYMENT.md).

```
│   ├─ public/             # Статические файлы
│   └─ src/                # Исходный код

    ├─ src/                # Исходный код

    │   ├─ controllers/    # Контроллеры
    │   ├─ models/         # Mongoose модели
    │   ├─ routes/         # Маршруты API
    │   └─ services/       # Сервисы для работы с внешними API
    └─ dist/               # Скомпилированный код (после сборки)
```
