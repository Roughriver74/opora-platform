## 1. Продуктовая суть

### Что такое ОПОРА

ОПОРА — конструктор для управления выездной работой. Не FSM-система, не SFA, не CRM. Конструктор: пользователь выбирает тип бизнеса при регистрации и получает готовую настроенную систему за 10 минут.

### Позиционирование

**Слоган**: "Контроль выездных — под ваш бизнес. За 10 минут."

| Конкуренты (FSM/SFA) | ОПОРА |
|---|---|
| Одна настройка под всех | Шаблон + кастомизация |
| Внедрение 1-4 недели | Настройка за 10 минут |
| 525-1700 ₽/чел/мес | Бесплатно до 5 человек |
| Native-приложение | PWA (без установки) |

### Ключевые отличия от конкурентов

1. **Шаблоны по типу бизнеса** — никто на рынке РФ не делает
2. **Freemium** — ни один конкурент не даёт бесплатный тариф
3. **Простота** — 10 минут до первого визита, без внедренцев
4. **PWA** — открыл ссылку и работаешь, без App Store

### Целевая аудитория

Любой бизнес с выездными сотрудниками (5-50 человек):
- Клининговые компании
- Сервис и монтаж оборудования
- Аудит и инспекции (франшизы, сети)
- Торговые представители

### Ядро ценности

1. **Контроль** — руководитель знает, что сотрудник был на месте и сделал работу
2. **Организация** — всё в одном месте: задачи, отчёты, фото, контакты

## 2. Система шаблонов

### Что такое шаблон

Шаблон — преднастроенный набор, который применяется при создании организации. Включает:

| Компонент | Что настраивает | Пример: "Клининг" |
|---|---|---|
| Формы визитов | Поля для заполнения | Тип уборки, площадь, средства |
| Статусы визитов | Жизненный цикл задачи | Назначена → В пути → Выполняется → Фотоотчёт → Завершена |
| Чек-листы | Пункты проверки | Полы, окна, санузел, кухня |
| Метрики дашборда | KPI на главном экране | Визитов/день, среднее время, % завершённых |
| Названия ролей | Косметика ролей | "Менеджер" / "Клинер" |

### Стартовый набор (4 + пустой)

**1. Клининг**
- Форма: тип уборки (генеральная/поддерживающая), площадь, использованные средства
- Чек-лист: полы, окна, санузел, кухня, мебель
- Статусы: Назначена → В пути → Выполняется → Фотоотчёт → Завершена
- Метрики: визитов/день, среднее время уборки, % завершённых

**2. Сервис и монтаж**
- Форма: тип работ, оборудование, использованные материалы, серийный номер
- Чек-лист: диагностика, ремонт/монтаж, проверка, подпись клиента
- Статусы: Заявка → Назначена → В пути → В работе → Завершена
- Метрики: заявок/день, среднее время выезда, % закрытых с первого раза

**3. Аудит и проверки**
- Форма: тип проверки, оценка (1-5), комментарий, критические отклонения
- Чек-лист: внешний вид, документация, стандарты, безопасность
- Статусы: Запланирована → В процессе → Завершена → Отчёт отправлен
- Метрики: проверок/неделя, средняя оценка, % критических отклонений

**4. Торговые представители**
- Форма: тип визита (плановый/внеплановый), заказ, остатки, конкуренты
- Чек-лист: выкладка, ценники, POS-материалы, просрочка
- Статусы: Запланирован → В пути → На точке → Отчёт → Завершён
- Метрики: визитов/день, сумма заказов, % выполнения плана

**5. Пустой шаблон**
- Минимальная форма: название, комментарий
- Статусы: Назначен → В работе → Завершён
- Пустой чек-лист
- Базовые метрики

### Технические детали

- Шаблоны хранятся как JSON-файлы в `app/templates/` (не в БД)
- При применении шаблон создаёт: FormTemplate + CustomSection + конфиг статусов + конфиг метрик
- После применения всё можно кастомизировать — шаблон это стартовая точка
- Модель Organization получает поле `template_id` для трекинга

## 3. Тарифная модель

### Три тарифа

| | Free | Pro | Business |
|---|---|---|---|
| **Цена** | 0 ₽ | 399 ₽/чел/мес | 699 ₽/чел/мес |
| **Пользователей** | до 5 | до 50 | безлимит |
| **Визитов/мес** | 100 | безлимит | безлимит |
| **Компаний** | 20 | безлимит | безлимит |
| **Шаблоны** | все 4 | все 4 | все 4 |
| **Кастомные формы** | 1 форма, до 5 полей | безлимит | безлимит |
| **Чек-листы** | базовые (из шаблона) | кастомные | кастомные |
| **Фотоотчёты** | до 3 фото/визит | до 20 фото/визит | безлимит |
| **Аналитика** | план/факт (базовая) | полная + экспорт CSV | полная + экспорт + API |
| **Интеграции** | — | Битрикс24 | Битрикс24 + REST API + вебхуки |
| **Брендинг** | лого ОПОРА | — | свой лого (white label) |
| **Поддержка** | FAQ | email | приоритетная |

### Логика ценообразования

- **Free** — щедрый, чтобы люди реально работали. 5 человек и 100 визитов — полноценная маленькая команда на месяц.
- **Pro 399 ₽** — дешевле всех конкурентов (Мобифорс 525 ₽, B2Field 690 ₽). Триггер перехода: лимит пользователей или нужны кастомные формы.
- **Business 699 ₽** — дешевле рынка, но даёт API и white label.

### Техническая реализация

Расширение существующего `plan_limits` в Organization.plan_limits (JSONB):

```json
{
  "max_users": 5,
  "max_companies": 20,
  "max_visits_per_month": 100,
  "max_forms": 1,
  "max_fields_per_form": 5,
  "max_photos_per_visit": 3,
  "custom_checklists": false,
  "analytics_export": false,
  "integrations": [],
  "api_enabled": false,
  "white_label": false
}
```

Ограничения — мягкие блоки с предложением апгрейда (не жёсткая стена).

## 4. PWA-клиент для выездного сотрудника

### Два вида интерфейса

Не отдельное приложение, а отдельный layout в React по роли:

- **Роль admin/org_admin** → полный интерфейс (текущий, с дашбордом руководителя)
- **Роль user** → упрощённый мобильный вид

### Интерфейс сотрудника

Главный экран — список визитов на сегодня:
- Прогресс-бар (3 из 5 визитов)
- Список визитов с временем, названием компании, статусом
- Кнопка "Начать визит" у текущего

### Флоу визита

1. **"Начать визит"** → браузер запрашивает геолокацию → координаты сохраняются как чекин (checkin_at, checkin_lat, checkin_lon)
2. **Заполнение формы/чек-листа** из шаблона
3. **Фото** через камеру PWA (`<input type="file" capture="camera">`) → привязка к визиту с координатами и временем
4. **"Завершить"** → координаты чекаута (checkout_at, checkout_lat, checkout_lon) → визит закрыт

### Геолокация

Не фоновый GPS-трекинг (невозможно надёжно в PWA), а **точечные чекины** — координаты фиксируются в момент "начал" и "завершил". Руководитель видит, что сотрудник был на месте.

### Фотоотчёты

- Камера через `<input type="file" capture="camera">`
- Фото сохраняется с метаданными: время, координаты
- Защита от загрузки старых фото: проверка, что время съёмки не старше 10 минут
- Хранение: локальная файловая система (MVP), S3 (позже)

### Новая модель VisitPhoto

```
VisitPhoto:
  - id (UUID)
  - visit_id (FK → Visit)
  - organization_id (FK → Organization)
  - file_path (String)
  - thumbnail_path (String, nullable)
  - latitude (Float, nullable)
  - longitude (Float, nullable)
  - taken_at (DateTime)
  - uploaded_at (DateTime)
  - file_size_bytes (Integer)
```

### Новые поля в Visit

```
Visit (добавить):
  - checkin_at (DateTime, nullable)
  - checkin_lat (Float, nullable)
  - checkin_lon (Float, nullable)
  - checkout_at (DateTime, nullable)
  - checkout_lat (Float, nullable)
  - checkout_lon (Float, nullable)
```

## 5. Онбординг

### Флоу нового пользователя

**Шаг 1**: Регистрация (email + пароль) — существующий экран

**Шаг 2**: "Как называется ваша компания?" — текстовое поле

**Шаг 3**: "Чем занимается ваша команда?" — выбор шаблона (4 карточки + "Своё")

**Шаг 4**: "Сколько выездных сотрудников?" — 1-5 / 6-20 / 21-50 / 50+

**Шаг 5**: Система создаёт организацию с применённым шаблоном

**Шаг 6**: Экран результата:
- "Готово! Вот что мы настроили:"
- Список применённых настроек (форма, чек-лист, статусы, дашборд)
- CTA: [Пригласить сотрудника] [Создать первый визит]

### Технические детали

- Онбординг — React-страница `OnboardingWizard`
- Новый эндпоинт: `POST /api/organizations/setup` с параметрами `{template_id, company_name, team_size}`
- Бэкенд: создаёт Organization + применяет шаблон (FormTemplate, CustomSection, статусы)
- Шаг 4 (количество сотрудников): если 50+ → показываем информацию о Pro/Business тарифах
- После онбординга пользователь перенаправляется на главную страницу

## 6. Дашборд руководителя

### Новые компоненты

**План/факт по сотрудникам:**
- Таблица: сотрудник | запланировано | выполнено | % | последний чекин
- Сортировка по % выполнения (отстающие сверху)

**Карта чекинов:**
- Leaflet (OpenStreetMap) — бесплатно, без API-ключей
- Точки последних чекинов сотрудников за сегодня
- По клику на точку — имя сотрудника, время, компания

**Лента фотоотчётов:**
- Последние N фото со всех визитов за сегодня
- Миниатюры, по клику — полноразмерное фото + инфо о визите

### Существующая аналитика

Дашборд аналитики (AnalyticsDashboardPage) уже есть: KPI карточки, LineChart, PieChart. Он остаётся, но дополняется план/факт виджетом.

## 7. Интеграции и API

### Уровень 1: Битрикс24 (тариф Pro)

Рефакторинг существующей интеграции в модуль:
- Пользователь вставляет webhook URL + ID смарт-процесса в настройках
- Маппинг полей: UI уже есть
- Двусторонняя синхронизация визитов и компаний
- Опционально — если не настроено, система работает автономно

### Уровень 2: Вебхуки (тариф Business)

Исходящие вебхуки — ОПОРА отправляет POST на URL клиента при событиях:
- `visit.created` — создан визит
- `visit.completed` — визит завершён
- `visit.checkin` — сотрудник чекинился
- `photo.uploaded` — загружено фото

Новая модель:
```
WebhookSubscription:
  - id (UUID)
  - organization_id (FK → Organization)
  - url (String)
  - events (ARRAY of String)
  - secret (String) — для подписи payload
  - is_active (Boolean)
  - created_at (DateTime)
```

### Уровень 3: REST API (тариф Business)

Публичный API с API-ключом:
- Визиты, компании, аналитика — чтение и запись
- Документация через Swagger (уже есть)

Новая модель:
```
ApiKey:
  - id (UUID)
  - organization_id (FK → Organization)
  - key_hash (String)
  - name (String)
  - permissions (ARRAY of String)
  - is_active (Boolean)
  - last_used_at (DateTime, nullable)
  - created_at (DateTime)
```

Rate limiting для внешних API: 100 запросов/минуту.

## 9. Фазы реализации

### Фаза 1: Ядро нового позиционирования

**Цель**: человек регистрируется → выбирает шаблон → система готова к работе.

Задачи:
1. Онбординг-визард (React: 4 шага + экран результата)
2. 4 JSON-шаблона + эндпоинт `POST /api/organizations/setup`
3. Чекин/чекаут с геолокацией (новые поля в Visit + API)
4. Фотоотчёты (модель VisitPhoto + загрузка + просмотр)
5. Расширение plan_limits (новые лимиты для 3 тарифов)

**Результат**: новый пользователь получает настроенную систему за 2 минуты.

### Фаза 2: Контроль для руководителя

**Цель**: руководитель видит, что происходит.

Задачи:
1. PWA-вид для сотрудника (упрощённый layout)
2. Дашборд план/факт (запланировано vs выполнено по сотрудникам)
3. Карта чекинов (Leaflet)
4. Лента фотоотчётов

**Результат**: руководитель за 30 секунд понимает: кто где, кто сделал, кто нет.

### Фаза 3: Монетизация и интеграции

**Цель**: продукт можно продавать.

Задачи:
1. Страница тарифов в UI (Free / Pro / Business)
2. Мягкие ограничения по тарифам с предложением апгрейда
3. Битрикс24 как модуль (рефакторинг)
4. Вебхуки (модель + настройка в UI + отправка)
5. API-ключи (модель + middleware)

**Результат**: монетизируемый продукт с интеграциями.

### Фаза 4: Полировка

**Цель**: продукт, который не стыдно показать.

Задачи:
1. Лендинг для неавторизованных (что такое ОПОРА, тарифы, шаблоны)
2. Push-уведомления (Web Push API)
3. Улучшение PWA (иконка, сплеш, offline-заглушка)
4. SEO-страницы под шаблоны ("управление клинингом", "контроль монтажников")

## 10. Метрики успеха

- **Регистрация → первый визит**: < 15 минут
- **Конверсия Free → Pro**: > 5% за первый месяц
- **Retention 30 дней**: > 40% (активные организации)
- **NPS**: > 30

## 11. Что НЕ делаем

- Native мобильное приложение (iOS/Android) — только PWA
- Фоновый GPS-трекинг — только точечные чекины
- Интеграция с 1С напрямую — только через API/вебхуки
- Telegram-бот — блокируется в РФ
- Маркетплейс шаблонов — преждевременно
- ИИ-функции — преждевременно
- Оптимизация маршрутов — не ядро продукта
