# Улучшения системы автокомплита

## 🎯 Проблемы, которые были решены

### 1. **Плохой поиск по частичным совпадениям**

- **Проблема**: Поиск работал только по точному совпадению или простому `LIKE` запросу
- **Решение**: Добавлены множественные стратегии поиска с приоритетами

### 2. **Проблема с копированием текста**

- **Проблема**: При вставке текста в поле автокомплит не находил нужные варианты
- **Решение**: Улучшена логика обработки вставленного текста и автоматического выбора

### 3. **Ограниченные поля поиска**

- **Проблема**: Поиск только по названию товара/компании
- **Решение**: Добавлен поиск по описанию, отраслям и другим полям

## 🚀 Новые возможности

### 1. **Множественные стратегии поиска на сервере**

#### Для товаров (`getProducts`):

1. **Поиск по ID** (если запрос - число)
2. **Точный поиск по названию**
3. **Поиск по частичному совпадению в названии**
4. **Поиск по описанию**
5. **Нечеткий поиск по словам**

#### Для компаний (`getCompanies`):

1. **Поиск по ID** (если запрос - число)
2. **Точный поиск по названию**
3. **Поиск по частичному совпадению в названии**
4. **Нечеткий поиск по словам** (включая отрасль)

### 2. **Умный поиск на клиенте**

#### Новый хук `useSmartSearch`:

- **5 стратегий поиска** с весами релевантности
- **История поиска** для улучшения результатов
- **Дебаунсированный поиск** для производительности
- **Ранжирование результатов** по релевантности

#### Стратегии умного поиска:

1. **Точное совпадение** (вес: 100)
2. **Начинается с** (вес: 80)
3. **Содержит** (вес: 60)
4. **Поиск по словам** (вес: 40)
5. **Нечеткий поиск** (вес: 20)

### 3. **Улучшенная обработка вставки текста**

#### В `AutocompleteInput`:

- **Обработчик `onPaste`** для вставки через буфер обмена
- **Автоматический поиск** при вставке текста
- **Улучшенная синхронизация** с выбранными опциями

#### В `useDynamicOptions`:

- **Трехуровневый поиск совпадений**:
  1. По `value` (ID)
  2. По `label` (название)
  3. По частичному совпадению

### 4. **Кэширование и производительность**

#### Кэш всех опций:

- **Локальное кэширование** всех загруженных опций
- **Умный поиск** по кэшированным данным
- **Обновление кэша** при новых запросах

#### Оптимизация запросов:

- **Приоритет кэша** перед серверными запросами
- **Дебаунсинг** для уменьшения нагрузки
- **Умное обновление** кэша

## 🔧 Как это работает

### 1. **Поиск товаров**

```
Запрос: "бетон"
1. Поиск по ID (если число)
2. Точный поиск: NAME = "бетон"
3. Частичный поиск: NAME LIKE "%бетон%"
4. Поиск по описанию: DESCRIPTION LIKE "%бетон%"
5. Нечеткий поиск по словам "бетон"
```

### 2. **Поиск компаний**

```
Запрос: "строительная"
1. Поиск по ID (если число)
2. Точный поиск: TITLE = "строительная"
3. Частичный поиск: TITLE LIKE "%строительная%"
4. Нечеткий поиск по словам в TITLE и INDUSTRY
```

### 3. **Умный поиск на клиенте**

```
Запрос: "ООО Строй"
1. Точное совпадение: "ООО Строй" (100 баллов)
2. Начинается с: "ООО Строй..." (80 баллов)
3. Содержит: "...ООО Строй..." (60 баллов)
4. По словам: "ООО" + "Строй" (40 баллов)
5. Нечеткий: "ООО" -> "ООО" (20 баллов)
```
