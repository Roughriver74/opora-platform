# Дашборд аналитики

## Обзор

Дашборд аналитики - это мощный инструмент для анализа эффективности работы с заявками, доступный только администраторам системы.

## Доступ к дашборду

- **URL**: `/dashboard`
- **Права доступа**: Только для пользователей с ролью `admin`
- **Навигация**: Доступен через главное меню и кнопку в админке

## Основные функции

### 1. Фильтрация данных

- **Период**: День, неделя, месяц, квартал, год
- **Диапазон дат**: Произвольный период
- **Статус заявок**: Все статусы, новые, в работе, завершенные, отмененные
- **Приоритет**: Все приоритеты, низкий, средний, высокий, срочный
- **Менеджер**: Фильтр по конкретному менеджеру

### 2. Основная статистика

- **Общее количество заявок** с показателем роста
- **Новые заявки** (требуют обработки)
- **Завершенные заявки** (успешно обработаны)
- **Заявки в работе** (обрабатываются)
- **Среднее время обработки**
- **Конверсия** (процент завершенных заявок)
- **Рост за месяц и неделю**

### 3. Инсайты и аналитика

- **Эффективность обработки** - процент успешно завершенных заявок
- **Средняя нагрузка** - количество заявок в день
- **Рост активности** - динамика по сравнению с предыдущим периодом
- **Топ исполнитель** - менеджер с наибольшим количеством заявок
- **Популярная форма** - форма с наибольшим количеством заявок
- **Активный клиент** - клиент с наибольшим количеством заявок

### 4. Графики и визуализация

- **Заявки по месяцам** - столбчатая диаграмма
- **Заявки по статусам** - круговая диаграмма
- **Заявки по приоритетам** - столбчатая диаграмма
- **Заявки по дням недели** - линейный график
- **Тренд времени обработки** - линейный график

### 5. Топ-листы

- **Топ форм** - рейтинг форм по количеству заявок
- **Топ менеджеров** - рейтинг менеджеров по количеству заявок
- **Топ клиентов** - рейтинг клиентов по количеству заявок
- **Топ товаров** - рейтинг товаров (планируется)
- **Топ статусов** - распределение по статусам
- **Топ приоритетов** - распределение по приоритетам

## Технические особенности

### Производительность

- Использует оптимизированные запросы к базе данных
- Денормализованные поля для быстрого доступа
- Предвычисленные поля (месяц, день недели)
- Кэширование данных на уровне API

### Безопасность

- Доступ только для администраторов
- Защищенные маршруты
- Валидация данных на сервере

### Архитектура

- **Frontend**: React + TypeScript + Material-UI
- **Графики**: Recharts
- **Даты**: MUI X Date Pickers
- **Backend**: Node.js + TypeScript + TypeORM
- **База данных**: PostgreSQL с оптимизированными индексами

## API Endpoints

### GET `/api/submissions/optimized/stats`

Получение статистики по заявкам с поддержкой фильтрации.

**Параметры запроса:**

- `dateFrom` - дата начала периода
- `dateTo` - дата окончания периода
- `assignedTo` - ID менеджера
- `userId` - ID пользователя
- `formId` - ID формы
- `status` - статус заявки
- `priority` - приоритет заявки

**Ответ:**

```json
{
  "success": true,
  "data": {
    "total": 150,
    "byStatus": [...],
    "byPriority": [...],
    "byMonth": [...],
    "byForm": [...],
    "byAssignedTo": [...],
    "byUser": [...],
    "byDayOfWeek": [...]
  }
}
```

## Планы развития

### Ближайшие улучшения

1. **Экспорт данных** - возможность экспорта отчетов в Excel/PDF
2. **Уведомления** - алерты при критических изменениях метрик
3. **Сравнение периодов** - сравнение текущего периода с предыдущим
4. **Детализация по товарам** - анализ популярности товаров
5. **Прогнозирование** - предсказание трендов на основе исторических данных

### Долгосрочные планы

1. **Интеграция с Битрикс24** - синхронизация данных для расширенной аналитики
2. **Машинное обучение** - автоматическое выявление аномалий и паттернов
3. **Мобильная версия** - адаптация для мобильных устройств
4. **Настраиваемые дашборды** - возможность создания персональных дашбордов

## Использование

1. **Вход в систему** под учетной записью администратора
2. **Переход на дашборд** через меню или кнопку в админке
3. **Настройка фильтров** для анализа нужного периода
4. **Анализ данных** через различные представления (статистика, графики, топы)
5. **Принятие решений** на основе полученных инсайтов

## Поддержка

При возникновении проблем с дашбордом:

1. Проверьте права доступа (роль должна быть `admin`)
2. Убедитесь, что в системе есть данные за выбранный период
3. Проверьте подключение к базе данных
4. Обратитесь к разработчикам для технической поддержки
