# Исправление поиска в заявках

## Проблема

При поиске в заявках фильтр по статусам не сбрасывался, что приводило к тому, что поиск работал только среди заявок с выбранным статусом, а не по всем заявкам.

**Пример проблемы:**

1. Пользователь выбирает статус "В работе"
2. Пользователь вводит поисковый запрос "ольгино"
3. Поиск ищет только среди заявок со статусом "В работе", а не по всем заявкам
4. Результат: "У вас пока нет заявок" (хотя заявки с "ольгино" могут быть в других статусах)

## Решение

Исправлена логика поиска в двух компонентах:

### 1. Пользовательские заявки (`MySubmissions/index.tsx`)

**До исправления:**

```typescript
const timeout = setTimeout(() => {
	handleFilterChange({ search: value })
}, 500)
```

**После исправления:**

```typescript
const timeout = setTimeout(() => {
	// При поиске сбрасываем фильтр по статусу, чтобы искать по всем заявкам
	const newFilters: Partial<SubmissionFilters> = { search: value }
	if (value.trim()) {
		// Если есть поисковый запрос, сбрасываем статус
		newFilters.status = undefined
	} else {
		// Если поиск пустой, возвращаем дефолтный фильтр
		newFilters.status = DEFAULT_STATUS_FILTER
	}
	handleFilterChange(newFilters)
}, 500)
```

### 2. Админские заявки (`SubmissionManagement/index.tsx`)

**До исправления:**

```typescript
const timeout = setTimeout(() => {
	handleFilterChange({ search: value })
}, 500)
```

**После исправления:**

```typescript
const timeout = setTimeout(() => {
	// При поиске сбрасываем фильтр по статусу, чтобы искать по всем заявкам
	const newFilters: Partial<SubmissionFilters> = { search: value }
	if (value.trim()) {
		// Если есть поисковый запрос, сбрасываем статус
		newFilters.status = undefined
	}
	handleFilterChange(newFilters)
}, 500)
```

## Логика работы

### При вводе поискового запроса:

1. ✅ **Сбрасывается фильтр по статусу** (`status = undefined`)
2. ✅ **Поиск работает по всем заявкам** независимо от статуса
3. ✅ **В выпадающем списке статуса** отображается "Все статусы"

### При очистке поиска:

1. ✅ **Возвращается дефолтный фильтр** (`status = DEFAULT_STATUS_FILTER`)
2. ✅ **Для пользователей:** показываются только "Новые" заявки
3. ✅ **Для админов:** показываются все заявки

## Результат

После исправления:

1. **Поиск работает корректно** - ищет по всем заявкам
2. **Фильтр по статусу автоматически сбрасывается** при вводе поискового запроса
3. **Интерфейс показывает правильное состояние** - "Все статусы" при активном поиске
4. **При очистке поиска** возвращается исходный фильтр

## Тестирование

Для проверки исправления:

1. **Выберите любой статус** в фильтре (например, "В работе")
2. **Введите поисковый запрос** (например, "ольгино")
3. **Проверьте, что:**
   - В выпадающем списке статуса отображается "Все статусы"
   - Поиск находит заявки независимо от их статуса
   - Результаты поиска корректны
