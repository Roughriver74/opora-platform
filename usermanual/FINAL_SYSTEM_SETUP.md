# Финальная настройка Beton CRM - Система ролей и форм

## Обзор системы

Система поддерживает две роли пользователей с разными интерфейсами:

### 👑 Администраторы (role: 'admin')
- Попадают в **админ-панель** с полным управлением системой
- Управление пользователями
- Создание и редактирование форм
- Просмотр заявок
- Интеграция с Битрикс24

### 👤 Обычные пользователи (role: 'user')
- Попадают на **динамическую форму заказа** из базы данных
- Видят активную форму, созданную админом
- Могут заполнять и отправлять заявки
- Минимальный интерфейс для удобства

## Как работает система

### 1. Администратор создает форму
1. Заходит в админ-панель → Формы → Новая форма
2. Создает поля (название, email, телефон, дата доставки, и т.д.)
3. Активирует форму

### 2. Пользователь заполняет форму
1. Входит в систему как обычный пользователь
2. Видит активную форму, созданную админом
3. Заполняет поля и отправляет заявку

### 3. Заявка обрабатывается
1. Заявка сохраняется в базе данных
2. Админ видит заявку в админ-панели
3. Заявка отправляется в Битрикс24 (если настроено)

## Компоненты системы

### Frontend
```
HomePage.tsx - Загружает активную форму из БД
BetoneForm/ - Динамический компонент формы
AdminPage.tsx - Админ-панель для управления
UserFormPage.tsx - Ссылка на HomePage для пользователей
```

### Backend
```
/api/forms - Получение форм (открыто для всех)
/api/forms/:id/submit - Отправка заявок (открыто)
/api/forms/* (CRUD) - Управление формами (только админы)
/api/users - Управление пользователями (только админы)
```

## Особенности

### Динамическая форма
- Форма строится из полей, созданных в админке
- Поддерживает различные типы полей (текст, email, дата, выбор)
- Валидация на основе настроек полей
- Прогресс заполнения и секционный режим

### Безопасность
- JWT токены с автоматическим обновлением
- Проверка ролей на уровне маршрутов
- Защищенные API endpoints для админов
- Открытые endpoints только для чтения форм и отправки заявок

### Навигация
- **Логотип/Главная** - ведет на форму (для пользователей) или админку (для админов)
- **Администрирование** - только для админов
- **Выйти** - для всех пользователей
- **Имя пользователя** - отображается в навигации

## API Endpoints

### Открытые (для всех авторизованных)
```
GET /api/forms - Список форм
GET /api/forms/:id - Конкретная форма
POST /api/forms/:id/submit - Отправка заявки
```

### Только для админов
```
POST /api/forms - Создание формы
PUT /api/forms/:id - Редактирование формы
DELETE /api/forms/:id - Удаление формы
/api/users/* - Управление пользователями
/api/submissions/* - Просмотр заявок
```
