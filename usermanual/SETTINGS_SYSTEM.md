# Система настроек с переключателями

Система позволяет администраторам включать/выключать различные функции приложения через веб-интерфейс.

## Доступные настройки

### Категория "Заявки" (submissions)

| Настройка                              | Тип     | По умолчанию        | Описание                                           |
| -------------------------------------- | ------- | ------------------- | -------------------------------------------------- |
| `submissions.enable_copying`           | boolean | true                | Показывать кнопку "Копировать" в списке заявок     |
| `submissions.copy_button_text`         | string  | "Копировать заявку" | Текст кнопки копирования                           |
| `submissions.allow_user_status_change` | boolean | true                | Разрешить пользователям менять статус своих заявок |
| `submissions.allow_user_edit`          | boolean | true                | Разрешить пользователям редактировать свои заявки  |

### Категория "Формы" (forms)

| Настройка                  | Тип    | По умолчанию | Описание                          |
| -------------------------- | ------ | ------------ | --------------------------------- |
| `forms.auto_save_interval` | number | 30000        | Интервал автосохранения форм (мс) |

### Категория "Интерфейс" (ui)

| Настройка       | Тип    | По умолчанию | Описание                     |
| --------------- | ------ | ------------ | ---------------------------- |
| `ui.theme_mode` | string | "light"      | Режим темы (light/dark/auto) |

### Категория "Система" (system)

| Настройка           | Тип     | По умолчанию | Описание                        |
| ------------------- | ------- | ------------ | ------------------------------- |
| `system.debug_mode` | boolean | false        | Режим отладки для разработчиков |

## Как использовать

### В админ-панели

1. Перейдите в **Администрирование → Настройки**
2. Найдите нужную категорию настроек
3. Используйте переключатели для boolean значений
4. Для текстовых и числовых значений используйте текстовые поля
5. Изменения применяются немедленно

### В коде (для получения настроек)

```typescript
import { settingsService } from '../services/settingsService'

// Получить значение настройки с fallback
const enableCopying = await settingsService.getSettingValue(
	'submissions.enable_copying',
	true
)

// Получить объект настройки целиком
const setting = await settingsService.getSetting('submissions.enable_copying')
```

## Влияние настроек на интерфейс

### Страница "Мои заявки"

- **submissions.enable_copying = false** → Кнопка "Копировать" скрыта
- **submissions.allow_user_edit = false** → Кнопка "Редактировать" скрыта
- **submissions.allow_user_status_change = false** → Выпадающий список статусов скрыт
- **submissions.copy_button_text** → Меняет текст кнопки копирования

## API для настроек

### Получение настроек (авторизованные пользователи)

```
GET /api/settings
GET /api/settings/category/{category}
GET /api/settings/{key}
```

### Управление настройками (только админы)

```
PUT /api/settings/{key}    - Обновить настройку
DELETE /api/settings/{key} - Удалить настройку
```

## Автоинициализация

При запуске сервера автоматически создаются настройки по умолчанию, если они отсутствуют в базе данных.

## Примеры использования

### Временное отключение копирования заявок

1. Зайдите в админку → Настройки
2. В категории "Заявки" найдите "Разрешить копирование заявок"
3. Переключите в OFF
4. Кнопки копирования исчезнут у всех пользователей

### Изменение текста кнопки

1. В категории "Заявки" найдите "Текст кнопки копирования"
2. Измените на "Дублировать"
3. Текст обновится для всех пользователей

### Запрет редактирования для обычных пользователей

1. Переключите "Разрешить пользователям редактировать свои заявки" в OFF
2. Обычные пользователи больше не смогут редактировать заявки
3. Админы по-прежнему видят все кнопки

Система настроек позволяет гибко управлять функциональностью приложения без изменения кода!
