# CRUD Пользователей - Beton CRM

## ✅ Что реализовано

### Backend (Полностью готов)

1. **User модель** (`server/src/models/User.ts`)
   - Email, пароль, роль (admin/user), имя, фамилия, телефон
   - Битрикс ID для интеграции
   - Статус (active/inactive)
   - Автоматическое хеширование паролей с bcrypt
   - Виртуальное поле fullName

2. **User контроллер** (`server/src/controllers/userController.ts`)
   - `getAllUsers` - список с пагинацией и фильтрацией
   - `getUserById` - получение по ID
   - `createUser` - создание с валидацией
   - `updateUser` - обновление с проверками уникальности
   - `deleteUser` - удаление с защитой последнего админа
   - `updateUserStatus` - изменение статуса
   - `syncWithBitrix` - синхронизация с Битрикс24

3. **User роуты** (`server/src/routes/userRoutes.ts`)
   - Все роуты защищены `requireAdmin`
   - Полный набор CRUD операций + синхронизация

4. **Главный админ** (Супер админ)
   - `superAdminLogin` - отдельный endpoint
   - Может логиниться при любом раскладе
   - Использует `SUPER_ADMIN_PASSWORD` или fallback на `ADMIN_PASSWORD`

5. **Интеграция с Битрикс24**
   - Получение пользователей из Битрикс24
   - Сопоставление по email и bitrix_id
   - Автоматическое создание и обновление

### Frontend (Частично готов)

1. **Типы пользователей** (`client/src/types/user.ts`)
   - Полный набор интерфейсов для User, CreateUserData, UpdateUserData
   - Типы для фильтрации, пагинации и ответов API

2. **UsersTable** (`client/src/components/admin/Users/UsersTable/`)
   - Таблица с пагинацией и фильтрацией
   - Поиск по имени и email
   - Фильтры по роли и статусу
   - Кнопки действий (редактировать, удалить)
   - Кнопка синхронизации с Битрикс24

3. **UserRow** - компонент строки пользователя
   - Отображение всех полей
   - Цветные чипы для ролей и статусов
   - Красивое форматирование даты

4. **useUsersTable** - хук для таблицы
   - Загрузка данных с дебаунсом поиска
   - Управление фильтрами и пагинацией
   - Обработка ошибок

5. **UsersPage** (`client/src/pages/admin/UsersPage.tsx`)
   - Основная страница управления пользователями
   - Диалоги подтверждения удаления и синхронизации
   - Snackbar уведомления

6. **Интеграция в AdminPage**
   - Новая вкладка "Пользователи" (первая)
   - Обновленная навигация между вкладками

## 🚀 API Endpoints

```
GET    /api/users              - Список пользователей (с фильтрами)
GET    /api/users/:id          - Пользователь по ID
POST   /api/users              - Создание пользователя
PUT    /api/users/:id          - Обновление пользователя
DELETE /api/users/:id          - Удаление пользователя
PATCH  /api/users/:id/status   - Изменение статуса
POST   /api/users/sync/bitrix  - Синхронизация с Битрикс24

POST   /api/auth/super-admin-login  - Вход главного админа
```

## 🔧 Главный админ

Создан механизм "главного администратора", который может войти при любых обстоятельствах:

### Переменные окружения:
```env
SUPER_ADMIN_PASSWORD=your-super-secret-password
# Или fallback на существующий
ADMIN_PASSWORD=your-admin-password
```

### Использование:
```bash
POST /api/auth/super-admin-login
{
  "password": "super_admin_password"
}
```

Главный админ получает:
- ID: "super_admin"
- Роль: "admin"
- Имя: "Главный Администратор"
- Полные права администратора

## 🔗 Интеграция с Битрикс24

### Синхронизация пользователей:
1. Получает пользователей из Битрикс24 через `user.get`
2. Сопоставляет по email или bitrix_id
3. Создает новых пользователей с случайными паролями
4. Обновляет существующих (если forceSync=true)
5. Возвращает статистику: созданных, обновленных, ошибок

### Поля синхронизации:
- EMAIL → email
- NAME → firstName  
- LAST_NAME → lastName
- WORK_PHONE → phone
- ID → bitrix_id
