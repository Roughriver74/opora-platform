# Настройка ролей пользователей в Beton CRM

## Обзор системы ролей

Система поддерживает две роли пользователей:
- **admin** - доступ к админ-панели для управления системой
- **user** - доступ к форме заказа бетона

## Поведение в зависимости от роли

### Администраторы (role: 'admin')
- Попадают на страницу администрирования
- Видят кнопку "Администрирование" в навигации
- Имеют доступ ко всем функциям системы

### Обычные пользователи (role: 'user')  
- Попадают на страницу с формой заказа
- Не видят кнопку "Администрирование"
- Могут только заполнять форму заказа

## Навигация и UI

### Общие элементы
- **Логотип:** Ведет на главную страницу
- **Имя пользователя:** Отображается в правом верхнем углу
- **Кнопка "Выйти":** Завершает сессию

### Для админов
- Кнопка "Администрирование" в навигации
- Доступ к управлению пользователями, формами и заявками

### Для обычных пользователей
- Простая форма заказа бетона
- Минимальный интерфейс для удобства

## Безопасность

- JWT токены с автоматическим обновлением
- Проверка ролей на уровне маршрутов
- Защищенные API endpoints для админов

## Тестирование

1. Войдите как администратор - должны увидеть админ-панель
2. Выйдите и войдите как обычный пользователь - должны увидеть форму
3. Проверьте, что кнопка "Администрирование" скрыта для обычных пользователей
4. Проверьте работу кнопки "Выйти"
