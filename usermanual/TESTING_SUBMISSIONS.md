## Сценарии тестирования

### Сценарий 1: Создание заявки пользователем

1. **Авторизация пользователя:**
   - Откройте http://localhost:3000
   - Войдите как `user@betonexpress.pro` / `user123`

2. **Создание заявки:**
   - На главной странице заполните форму заказа бетона
   - Нажмите "Отправить заявку"
   - Проверьте сообщение об успешной отправке

3. **Просмотр заявки:**
   - Нажмите "Мои заявки" в навигации
   - Убедитесь, что заявка появилась в списке
   - Кликните на иконку просмотра для деталей

### Сценарий 2: Управление заявками администратором

1. **Авторизация администратора:**
   - Выйдите из системы
   - Войдите как `crm@betonexpress.pro` / `Sacre.net13`

2. **Просмотр всех заявок:**
   - Нажмите "Администрирование"
   - Перейдите на вкладку "Заявки"
   - Убедитесь, что видны все заявки системы

3. **Фильтрация заявок:**
   - Используйте поиск по номеру заявки
   - Фильтруйте по статусу
   - Попробуйте разные фильтры

4. **Изменение статуса:**
   - Выберите заявку
   - Измените статус через выпадающий список
   - Проверьте, что изменение сохранилось

5. **Назначение заявки:**
   - Назначьте заявку на пользователя
   - Проверьте изменение в списке

6. **Просмотр деталей:**
   - Кликните на иконку просмотра
   - Проверьте отображение:
     - Основной информации
     - Данных формы
     - Истории изменений

### Сценарий 3: Анонимная заявка

1. **Выход из системы:**
   - Выйдите из аккаунта администратора

2. **Отправка анонимной заявки:**
   - На странице входа заполните форму
   - Отправьте заявку без авторизации
   - Проверьте успешную отправку

3. **Проверка в админ-панели:**
   - Войдите как администратор
   - Найдите анонимную заявку (без пользователя)

## Проверка API endpoints

### Публичные endpoints:
```bash
# Отправка заявки
curl -X POST http://localhost:5000/api/submissions/submit \
  -H "Content-Type: application/json" \
  -d '{
    "formId": "FORM_ID",
    "formData": {
      "name": "Тест",
      "phone": "+7999999999"
    }
  }'
```

### Защищенные endpoints (требуют авторизации):
```bash
# Получение заявок (нужен JWT токен)
curl -X GET http://localhost:5000/api/submissions \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"

# Обновление статуса
curl -X PATCH http://localhost:5000/api/submissions/SUBMISSION_ID/status \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"status": "in_progress", "comment": "Взято в работу"}'
```

## Возможные проблемы и решения

### Проблема: Заявки не создаются
**Решение:** Проверьте:
- Подключение к MongoDB
- Наличие активных форм в системе
- Корректность данных формы

### Проблема: Ошибки авторизации
**Решение:** Проверьте:
- JWT токены
- Роли пользователей
- Middleware авторизации

### Проблема: Битрикс24 не работает
**Решение:** 
- Заявки сохраняются в БД независимо от Битрикс24
- Проверьте настройки интеграции
- Статус синхронизации отображается в заявке
