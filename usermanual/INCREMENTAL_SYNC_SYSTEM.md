# Система инкрементальной синхронизации Elasticsearch

## Обзор

Реализована новая система инкрементальной синхронизации Elasticsearch, которая решает проблему окна недоступности при полной пересоздании индексов. Система использует паттерн "Alias Swap" для безопасного обновления данных без прерывания работы поиска.

## Проблемы старой системы

- ❌ Полное удаление индекса создавало окно недоступности на 2-3 минуты
- ❌ Риск потери данных при сбое синхронизации
- ❌ Невозможность поиска во время синхронизации
- ❌ Неэффективное использование ресурсов

## Решение

### 1. Система отслеживания изменений

**Сущность SyncMetadata** (`SyncMetadata.entity.ts`):

- Отслеживает время последней синхронизации для каждого типа данных
- Хранит статистику синхронизации (количество записей, ошибки)
- Поддерживает статусы: `pending`, `in_progress`, `completed`, `failed`

**Сервис SyncMetadataService** (`syncMetadataService.ts`):

- Управление метаданными синхронизации
- Определение необходимости полной синхронизации
- Отслеживание прогресса и ошибок

### 2. Безопасное переключение индексов

**Alias Swap Pattern**:

- Создание временного индекса параллельно с основным
- Загрузка данных в временный индекс
- Атомарное переключение алиаса на новый индекс
- Удаление старого индекса после успешного переключения

**Методы ElasticsearchService**:

- `createTemporaryIndex()` - создание временного индекса
- `switchAlias()` - переключение алиаса
- `deleteOldIndex()` - удаление старого индекса
- `initializeAlias()` - инициализация алиаса

### 3. Инкрементальная синхронизация

**IncrementalSyncService** (`incrementalSyncService.ts`):

- Инкрементальная синхронизация продуктов, компаний и заявок
- Автоматическое определение необходимости полной синхронизации
- Пакетная обработка данных для оптимизации производительности
- Поддержка различных стратегий синхронизации

## Стратегии синхронизации

### 1. Полная синхронизация (Full Sync)

- Выполняется при первом запуске или принудительно
- Создает временный индекс
- Загружает все данные
- Переключает алиас атомарно
- Удаляет старый индекс

### 2. Инкрементальная синхронизация (Incremental Sync)

- Обновляет только измененные данные
- Использует `bulkUpsert` для обновления существующих записей
- Не создает временный индекс
- Быстрее полной синхронизации

### 3. Автоматическое определение

- Проверяет время последней полной синхронизации
- Если прошло больше `maxAgeHours` (по умолчанию 24 часа) - выполняет полную синхронизацию
- Иначе - инкрементальную

## Конфигурация

### Параметры синхронизации

- `forceFullSync`: Принудительная полная синхронизация
- `batchSize`: Размер пакета для обработки (по умолчанию 100)
- `maxAgeHours`: Максимальный возраст для инкрементальной синхронизации (по умолчанию 24 часа)
