# 🚀 Руководство по оптимизации Elasticsearch

## 📋 Обзор улучшений

В проекте реализованы следующие оптимизации Elasticsearch на основе лучших практик:

### 1. 🧠 Улучшенные анализаторы

- **`product_search`** - основной анализатор для поиска товаров с синонимами и edgeNGram
- **`autocomplete`** - специализированный анализатор для автокомплита
- **`exact_match`** - анализатор для точного поиска
- **`russian`** - улучшенный анализатор для русского языка

### 2. 🔗 Синонимы для товаров

Автоматическое расширение поиска по синонимам:

- `бетон,цемент,раствор`
- `песок,песчаный,песчаная`
- `щебень,гравий,камень`
- `арматура,металл,сталь`
- `доставка,транспорт,перевозка`
- `м300,м-300,марка 300`
- `м400,м-400,марка 400`
- `м500,м-500,марка 500`
- `фундамент,основание,база`
- `строительство,стройка,возведение`

### 3. 🎯 Function Score для релевантности

- Бустинг новых документов (созданных за последние 30 дней)
- Бустинг обновленных документов (обновленных за последние 7 дней)
- Улучшенное ранжирование результатов

### 4. 🔍 Улучшенная обработка опечаток

- `fuzziness: 'AUTO'` - автоматический подбор допустимых опечаток
- `fuzzy_transpositions: true` - учет перестановок символов
- `prefix_length: 1` - минимальная длина префикса
- `max_expansions: 50` - ограничение расширений

### 5. 📊 Оптимизированные запросы

- **Multi-match** с бустингом полей
- **Match phrase** для точных фраз
- **Wildcard** для частичных совпадений
- **Prefix** для поиска по началу слова

## 📈 Использование в коде

### Поиск с оптимизациями

```typescript
// Поиск с обработкой опечаток
const results = await elasticsearchService.search({
	query: 'бетн м300', // Опечатка в "бетон"
	type: 'product',
	limit: 10,
	fuzzy: true, // Включает обработку опечаток
	includeHighlights: true,
})

// Автокомплит
const suggestions = await elasticsearchService.suggest('бет', 'product')
```

### API эндпоинты

Все существующие API эндпоинты автоматически используют оптимизации:

- `POST /api/search/products` - поиск товаров
- `POST /api/search/companies` - поиск компаний
- `POST /api/search/contacts` - поиск контактов
- `POST /api/search/submissions` - поиск заявок
- `POST /api/search/suggest` - автокомплит

## 🔧 Настройка и кастомизация

### Добавление новых синонимов

Отредактируйте файл `elasticsearchService.ts`:

```typescript
product_synonyms: {
  type: 'synonym',
  synonyms: [
    'бетон,цемент,раствор',
    'песок,песчаный,песчаная',
    // Добавьте новые синонимы здесь
    'новый_синоним,альтернатива',
  ],
},
```

### Настройка бустинга полей

В методе `search()` измените веса полей:

```typescript
fields: [
  'name^4',                    // Название - самый высокий приоритет
  'name.autocomplete^3.5',     // Автокомплит поле
  'description^2.5',           // Описание
  'searchableText^2',          // Поисковый текст
  'industry^2',                // Отрасль
  // Добавьте новые поля с весами
],
```

### Настройка function_score

Измените функции бустинга в методе `search()`:

```typescript
functions: [
  {
    filter: {
      range: {
        createdAt: {
          gte: 'now-30d', // Измените период
        },
      },
    },
    weight: 1.2, // Измените вес
  },
],
```

## 🚨 Устранение неполадок

### Проблема: Поиск не работает

1. Проверьте здоровье Elasticsearch:

   ```bash
   curl http://localhost:9200/_cluster/health
   ```

2. Переиндексируйте данные:
   ```bash
   npm run ts-node src/scripts/reindexWithOptimizedSettings.ts
   ```

### Проблема: Медленный поиск

1. Проверьте размер индекса:

   ```bash
   curl http://localhost:9200/beton_crm_search/_stats
   ```

2. Оптимизируйте запросы, уменьшив `max_expansions`

### Проблема: Нет результатов с опечатками

1. Убедитесь, что `fuzzy: true` передается в запрос
2. Проверьте настройки `fuzziness` в анализаторе

## 🔧 Исправления фронтенда

### Проблема

Фронтенд не передавал параметр `fuzzy: true` в API запросы, что приводило к неэффективному поиску без обработки опечаток.

### Решение

1. **Обновлен FormFieldService** - добавлен `fuzzy: true` во все методы поиска
2. **Создан оптимизированный хук useOptimizedSearch** с:
   - Debouncing (300ms) для уменьшения количества запросов
   - Кэшированием результатов
   - Умной обработкой запросов
   - Лучшей обработкой ошибок

### Результат

- ✅ Поиск с опечатками теперь работает корректно
- ✅ Уменьшено количество запросов к серверу
- ✅ Улучшена производительность фронтенда
- ✅ Лучший UX с кэшированием результатов
