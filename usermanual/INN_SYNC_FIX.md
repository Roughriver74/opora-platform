# Исправление синхронизации ИНН в Elasticsearch

## Проблема

ИНН компаний не отображались в результатах поиска Elasticsearch на дев сервере. При проверке выяснилось, что в индексе есть только наименования компаний, но нет ИНН.

## Причина

Проблема была в том, что в `searchSyncService.ts` использовался метод `getAllCompanies()`, который запрашивает поле `RQ_INN` из основной таблицы компаний Bitrix24. Однако в этой таблице поле `RQ_INN` обычно пустое, так как ИНН хранится в отдельной таблице реквизитов.

## Решение

### 1. Исправлен метод синхронизации компаний

В файле `server/src/services/searchSyncService.ts`:

**Было:**

```typescript
const companies = await bitrix24Service.getAllCompanies()
```

**Стало:**

```typescript
const companies = await bitrix24Service.getAllCompaniesWithRequisites()
```

### 2. Добавлен ИНН в поисковый текст

В методе `buildSearchableText()` добавлено поле ИНН:

```typescript
const searchableFields = [
	obj.NAME || obj.TITLE || '',
	obj.DESCRIPTION || obj.COMMENTS || '',
	obj.INDUSTRY || '',
	obj.ADDRESS || '',
	obj.PHONE?.[0]?.VALUE || '',
	obj.EMAIL?.[0]?.VALUE || '',
	obj.RQ_INN || '', // Добавляем ИНН в поисковый текст
]
```

## Разница между методами

### `getAllCompanies()`

- Запрашивает данные только из основной таблицы компаний
- Поле `RQ_INN` в этой таблице обычно пустое
- Быстрее выполняется, но не содержит реквизиты

### `getAllCompaniesWithRequisites()`

- Получает данные из основной таблицы компаний
- Дополнительно загружает реквизиты из таблицы `crm.requisite`
- Извлекает ИНН из активных реквизитов
- Медленнее, но содержит полную информацию

## Проверка результатов

После исправления:

1. **Синхронизация**: ИНН будет правильно загружаться из реквизитов
2. **Поиск**: ИНН будет включен в поисковый текст
3. **Отображение**: ИНН будет отображаться в результатах поиска
4. **Поиск по ИНН**: Можно будет искать компании по их ИНН

## Рекомендации

1. **Пересинхронизация**: После внесения изменений необходимо пересинхронизировать данные
2. **Мониторинг**: Следить за производительностью, так как загрузка реквизитов занимает больше времени
3. **Кэширование**: Рассмотреть возможность кэширования реквизитов для ускорения последующих синхронизаций
