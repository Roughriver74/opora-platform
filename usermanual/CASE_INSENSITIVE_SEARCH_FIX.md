# Исправление поиска без учета регистра

## Проблема

Поиск в заявках был чувствителен к регистру. Когда пользователь искал "мшков" с маленькой буквы, а в системе есть "Мшков" с большой буквы, поиск не находил результат.

**Пример проблемы:**

- Пользователь ищет: "мшков"
- В системе есть: "Мшков"
- Результат: "У вас пока нет заявок" (хотя заявка существует)

## Причина

Elasticsearch запросы использовали оригинальный регистр поискового запроса без приведения к нижнему регистру, хотя анализаторы настроены на обработку текста в нижнем регистре.

## Решение

Исправлены все типы поисковых запросов в `elasticsearchService.ts`:

### 1. Match Phrase запросы

```typescript
// ДО
{
    match_phrase: {
        name: {
            query: originalQuery,
            boost: 5,
        },
    },
}

// ПОСЛЕ
{
    match_phrase: {
        name: {
            query: originalQuery.toLowerCase(), // Приводим к нижнему регистру
            boost: 5,
        },
    },
}
```

### 2. Match запросы

```typescript
// ДО
{
    match: {
        'name.exact': {
            query: originalQuery,
            boost: 4.5,
        },
    },
}

// ПОСЛЕ
{
    match: {
        'name.exact': {
            query: originalQuery.toLowerCase(), // Приводим к нижнему регистру
            boost: 4.5,
        },
    },
}
```

### 3. Multi-match запросы

```typescript
// ДО
multi_match: {
    query: variant,
    fields: [...],
    ...
}

// ПОСЛЕ
multi_match: {
    query: variant.toLowerCase(), // Приводим к нижнему регистру
    fields: [...],
    ...
}
```

### 4. Wildcard запросы

```typescript
// ДО
{
    wildcard: {
        name: {
            value: `*${normalizedQuery}*`,
            boost: 1.5,
        },
    },
}

// ПОСЛЕ
{
    wildcard: {
        name: {
            value: `*${normalizedQuery.toLowerCase()}*`, // Приводим к нижнему регистру
            boost: 1.5,
        },
    },
}
```

## Что было исправлено

### В файле `elasticsearchService.ts`:

1. **Match phrase запросы** - добавлено `.toLowerCase()`
2. **Match запросы** (exact и autocomplete) - добавлено `.toLowerCase()`
3. **Multi-match запросы** - добавлено `.toLowerCase()`
4. **Wildcard запросы** - добавлено `.toLowerCase()`
5. **Поиск без пробелов** - добавлено `.toLowerCase()`

## Результат

После исправления:

1. **Поиск "мшков"** найдет "Мшков"
2. **Поиск "МШКОВ"** найдет "мшков"
3. **Поиск "Мшков"** найдет "мшков"
4. **Любой регистр** будет работать корректно

## Тестирование

Для проверки исправления:

1. **Введите поисковый запрос с маленькой буквы** (например, "мшков")
2. **Проверьте, что находятся заявки** с названиями в любом регистре
3. **Попробуйте разные варианты регистра** для одного и того же слова

## Техническая информация

### Анализаторы Elasticsearch

Анализаторы уже были настроены правильно:

- `product_search` - использует фильтр `lowercase`
- `exact_match` - использует фильтр `lowercase`
- `autocomplete` - использует фильтр `lowercase`

### Проблема была в запросах

Запросы передавали оригинальный регистр, не используя возможности анализаторов.
