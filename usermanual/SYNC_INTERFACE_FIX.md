# Исправление интерфейса синхронизации

## Ответы на вопросы

### 1. Почему не поменялась информация в интерфейсе?

**Проблема:** В клиентском сервисе `syncService.ts` отсутствовала опция "Каждые 30 минут".

**Решение:** Добавлена опция "Каждые 30 минут" в список доступных расписаний:

```typescript
getAvailableSchedules(): { [key: string]: string } {
    return {
        'Каждые 30 минут': '*/30 * * * *', // ← ДОБАВЛЕНО
        'Каждый час': '0 * * * *',
        'Каждые 3 часа': '0 */3 * * *',
        'Каждые 6 часов': '0 */6 * * *',
        // ... остальные опции
    }
}
```

### 2. Запускает ли кнопка "СИНХРОНИЗИРОВАТЬ" корректный скрипт?

**Да!** Кнопка "СИНХРОНИЗИРОВАТЬ" теперь запускает правильный скрипт:

1. **Кнопка** → `syncService.startSync()`
2. **API запрос** → `POST /api/sync/start`
3. **Backend** → `syncController.startSync()`
4. **Сервис** → `searchSyncService.syncAllData()`
5. **Результат** → Синхронизирует продукты, компании (с ИНН) и заявки
