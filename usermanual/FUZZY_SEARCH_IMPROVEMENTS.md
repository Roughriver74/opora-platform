# Улучшения нечеткого поиска для обработки опечаток

## Проблема

Поиск "мошков" не находил "Мшков" из-за недостаточно гибких настроек нечеткого поиска. Elasticsearch не мог обработать случай, когда в поисковом запросе пропущена буква.

**Пример проблемы:**

- Пользователь ищет: "мошков"
- В системе есть: "Мшков" (без буквы "о")
- Результат: не находил заявку

## Причина

Настройки нечеткого поиска были недостаточно гибкими:

- `fuzziness: 'AUTO'` - автоматический расчет расстояния Левенштейна
- `prefix_length: 1` - первые символы должны точно совпадать
- `max_expansions: 50` - ограниченное количество вариантов

## Решение

Улучшены настройки нечеткого поиска в `elasticsearchService.ts`:

### 1. Увеличен fuzziness

```typescript
// ДО
fuzziness: fuzzy ? 'AUTO' : 0,

// ПОСЛЕ
fuzziness: fuzzy ? 2 : 0, // Увеличиваем fuzziness до 2 для лучшего поиска опечаток
```

### 2. Убран prefix_length

```typescript
// ДО
prefix_length: 1,

// ПОСЛЕ
prefix_length: 0, // Убираем prefix_length для лучшего поиска в начале слова
```

### 3. Увеличен max_expansions

```typescript
// ДО
max_expansions: 50,

// ПОСЛЕ
max_expansions: 100, // Увеличиваем max_expansions
```

### 4. Добавлен дополнительный fuzzy запрос

```typescript
// Дополнительный нечеткий поиск для опечаток в начале слова
{
    fuzzy: {
        name: {
            value: originalQuery.toLowerCase(),
            fuzziness: 2,
            boost: 0.8,
        },
    },
},
```

## Что было улучшено

### В файле `elasticsearchService.ts`:

1. **Multi-match запросы** - улучшены настройки нечеткого поиска
2. **Добавлен fuzzy запрос** - дополнительный поиск с учетом опечаток
3. **Увеличен fuzziness** - с 'AUTO' до 2 для лучшего поиска опечаток
4. **Убран prefix_length** - теперь поиск работает и в начале слова
5. **Увеличен max_expansions** - больше вариантов для поиска

## Результат

- Заявки с частичным совпадением

## Техническая информация

### Fuzziness = 2 означает:

- **0** - точное совпадение
- **1** - одна замена, вставка или удаление символа
- **2** - две замены, вставки или удаления символов

### Примеры работы:

- "мошков" → "мшков" (удаление "о") - расстояние = 1 ✅
- "мошков" → "мошков" (точное совпадение) - расстояние = 0 ✅
- "мошков" → "мошково" (добавление "о") - расстояние = 1 ✅

### Prefix_length = 0 означает:

- Поиск работает с самого начала слова
- Не требует точного совпадения первых символов
