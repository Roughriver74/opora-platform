# Система управления заявками Beton CRM

## Обзор системы

Система управления заявками обеспечивает полный жизненный цикл заявок от создания до завершения с отслеживанием истории изменений и интеграцией с Битрикс24.

## Архитектура

### Серверная часть

#### Модели данных

**1. Submission (Заявка)**
```typescript
interface ISubmission {
  _id: string;
  submissionNumber: string; // Уникальный номер (YYYYMMDD0001)
  formId: ObjectId; // Ссылка на форму
  userId?: ObjectId; // Пользователь (может быть анонимная заявка)
  formData: Record<string, any>; // Данные формы
  status: 'new' | 'in_progress' | 'completed' | 'cancelled' | 'on_hold';
  priority: 'low' | 'medium' | 'high' | 'urgent';
  
  // Битрикс24 интеграция
  bitrixDealId?: string;
  bitrixSyncStatus: 'pending' | 'synced' | 'failed';
  bitrixSyncError?: string;
  
  // Назначение и заметки
  assignedTo?: ObjectId; // Назначенный менеджер
  notes?: string;
  tags: string[];
  
  createdAt: Date;
  updatedAt: Date;
}
```

**2. SubmissionHistory (История изменений)**
```typescript
interface ISubmissionHistory {
  _id: string;
  submissionId: ObjectId;
  changedBy: ObjectId; // Кто изменил
  changeType: 'status_change' | 'priority_change' | 'assignment' | 'data_update' | 'note_added' | 'tag_added' | 'tag_removed';
  
  oldValue?: any;
  newValue?: any;
  fieldName?: string;
  description: string; // Автоматическое описание
  comment?: string; // Комментарий пользователя
  
  changedAt: Date;
  ipAddress?: string;
  userAgent?: string;
}
```

#### API Endpoints

**Публичные:**
- `POST /api/submissions/submit` - Отправка заявки (авторизованная или анонимная)

**Для пользователей:**
- `GET /api/submissions/my/submissions` - Получение своих заявок

**Для администраторов:**
- `GET /api/submissions` - Получение всех заявок с фильтрацией
- `GET /api/submissions/:id` - Получение заявки с историей
- `PATCH /api/submissions/:id/status` - Обновление статуса
- `PATCH /api/submissions/:id/assign` - Назначение заявки

#### Контроллеры

**submissionController.ts:**
- `submitForm` - Создание заявки с синхронизацией в Битрикс24
- `getAllSubmissions` - Получение заявок с фильтрацией и пагинацией
- `getSubmissionById` - Получение заявки с историей
- `updateSubmissionStatus` - Обновление статуса с записью в историю
- `assignSubmission` - Назначение заявки с записью в историю

## Функциональность

### Создание заявки

1. **Отправка формы:**
   - Валидация обязательных полей
   - Создание записи в БД с уникальным номером
   - Попытка синхронизации с Битрикс24
   - Создание записи в истории (для авторизованных пользователей)

2. **Генерация номера заявки:**
   - Формат: YYYYMMDD0001 (год+месяц+день+порядковый номер)
   - Автоматическая генерация при сохранении

### Управление заявками (Администраторы)

1. **Фильтрация и поиск:**
   - По статусу, приоритету, назначенному менеджеру
   - Поиск по номеру заявки, заметкам, тегам
   - Сортировка по различным полям

2. **Изменение статуса:**
   - Выбор из выпадающего списка
   - Автоматическая запись в историю
   - Возможность добавить комментарий

3. **Назначение заявки:**
   - Выбор менеджера из списка пользователей
   - Запись в историю с именами пользователей

4. **Просмотр деталей:**
   - Полная информация о заявке
   - Данные формы
   - История изменений с временными метками

### Просмотр заявок (Пользователи)

1. **Список заявок:**
   - Только свои заявки
   - Статус и приоритет
   - Дата создания

2. **Детали заявки:**
   - Информация о статусе
   - Назначенный менеджер
   - История изменений
   - ID сделки в CRM

## Статусы заявок

- **new** (Новая) - Только что созданная заявка
- **in_progress** (В работе) - Заявка взята в работу
- **completed** (Завершена) - Заявка выполнена
- **cancelled** (Отменена) - Заявка отменена
- **on_hold** (Приостановлена) - Работа приостановлена

## Приоритеты

- **low** (Низкий) - Обычная заявка
- **medium** (Средний) - Стандартный приоритет
- **high** (Высокий) - Важная заявка
- **urgent** (Срочный) - Требует немедленного внимания

## Интеграция с Битрикс24

1. **Синхронизация:**
   - Автоматическое создание сделки при отправке заявки
   - Сохранение ID сделки в заявке
   - Обработка ошибок синхронизации

2. **Статусы синхронизации:**
   - **pending** - Ожидает синхронизации
   - **synced** - Успешно синхронизировано
   - **failed** - Ошибка синхронизации

## Безопасность

1. **Авторизация:**
   - Публичная отправка заявок (с возможностью авторизации)
   - Просмотр только своих заявок для пользователей
   - Полный доступ для администраторов

2. **Middleware:**
   - `authMiddleware` - Проверка авторизации
   - `adminMiddleware` - Проверка прав администратора

## Навигация

1. **Пользователи:**
   - Кнопка "Мои заявки" в навигации
   - Доступ к `/my-submissions`

2. **Администраторы:**
   - Вкладка "Заявки" в админ-панели
   - Полное управление всеми заявками

## Использование

### Для пользователей:
1. Заполните форму на главной странице
2. Перейдите в "Мои заявки" для отслеживания статуса
3. Просматривайте детали и историю изменений

### Для администраторов:
1. Откройте админ-панель → вкладка "Заявки"
2. Используйте фильтры для поиска нужных заявок
3. Изменяйте статус и назначайте менеджеров
4. Просматривайте детали и историю изменений
