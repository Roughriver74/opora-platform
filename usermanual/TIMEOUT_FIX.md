# ⏰ Исправление таймаута синхронизации

## ✅ Проблема решена!

Исправлена ошибка таймаута при синхронизации. Теперь синхронизация работает корректно с увеличенными таймаутами и улучшенной обработкой ошибок.

## 🔧 Что было исправлено

### 1. **Ошибка таймаута 60 секунд**

#### ❌ Проблема:

```
[Warning] API синхронизации недоступен: – Vs {stack: "...", message: "timeout of 60000ms exceeded", name: "AxiosError", code: "ECONNABORTED", ...}
```

#### ✅ Решение:

- **Увеличены таймауты** для запросов синхронизации:
  - Инкрементальная синхронизация: **5 минут** (300,000ms)
  - Полная синхронизация: **10 минут** (600,000ms)
- **Увеличен batchSize** с 100 до 500 для более быстрой обработки
- **Улучшена обработка ошибок** с детальными сообщениями

### 2. **Улучшенная обработка ошибок**

#### ✅ Новые сообщения об ошибках:

- **Таймаут**: "Синхронизация занимает слишком много времени. Попробуйте позже или используйте кнопку 'Принудительно' для полной синхронизации."
- **Ошибка сервера**: "Ошибка сервера при синхронизации. Проверьте логи сервера."
- **API не найден**: "API синхронизации не найден. Проверьте конфигурацию сервера."

## 🚀 Оптимизации производительности

### ✅ Увеличенный batchSize:

```typescript
// Было: batchSize: 100
// Стало: batchSize: 500
const response = await api.post(
	'/api/incremental-sync/all',
	{
		forceFullSync: false,
		batchSize: 500, // Увеличиваем batchSize для более быстрой синхронизации
		maxAgeHours: 24,
	},
	{
		timeout: 300000, // 5 минут для синхронизации
	}
)
```

### ✅ Адаптивные таймауты:

```typescript
// Инкрементальная синхронизация: 5 минут
timeout: 300000

// Полная синхронизация: 10 минут
timeout: 600000

// Адаптивный таймаут в runIncrementalSync
const timeout = options.forceFullSync ? 600000 : 300000
```
